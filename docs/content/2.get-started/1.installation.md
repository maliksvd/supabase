The [@nuxtjs/supabase](https://github.com/nuxt-modules/supabase) module offers a robust, first-class integration of Supabase with Nuxt, making it easier for developers to harness Supabase's powerful backend features—such as authentication, real-time databases, and APIs—within their Nuxt applications, while ensuring optimal compatibility and performance.

::callout{icon="i-heroicons-information-circle-16-solid"}
Explore the [Nuxt 3](https://v3.nuxtjs.org) documentation and [Supabase](https://supabase.com) to learn more.
::

## CLI

Run this command to add `@nuxtjs/supabase` module to your project using the [Nuxt CLI](https://v3.nuxtjs.org/docs/getting-started/installation#installation-cli).

```bash [Terminal]
npx nuxi@latest module add supabase
```

## Manual

Install `@nuxtjs/supabase` module manually via your preferred package manager.

::code-group

```bash [pnpm]
pnpm add @nuxtjs/supabase
```

```bash [yarn]
yarn add @nuxtjs/supabase
```

```bash [npm]
npm install @nuxtjs/supabase
```

::

Add `@nuxtjs/supabase` to the `modules` section of `nuxt.config.ts`:

```ts [nuxt.config.ts]
export default defineNuxtConfig({
  modules: ["@nuxtjs/supabase"],
});
```

Then, add `SUPABASE_URL` and `SUPABASE_KEY` to the `.env` to give access to the Supabase API.

```bash [.env]
SUPABASE_URL="https://example.supabase.co"
SUPABASE_KEY="<your_key>"
```

::callout{icon="i-heroicons-light-bulb"}
Alternatively, you can prefix the env variables with `NUXT_PUBLIC_` in order to use runtimeConfig.
::
