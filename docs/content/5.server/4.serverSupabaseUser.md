::callout{icon="i-heroicons-info-circle-20"}
Before proceeding, make sure you're familiar with Nitro, the server engine behind Nuxt.
::

The serverSupabaseUser function works similarly to the [useSupabaseUser](/usage/composables/usesupabaseuser) composable but is used in [server routes](https://nuxt.com/docs/guide/directory-structure/server#server-routes).

## How to use

1. Define your server route and import the `serverSupabaseUser` from `#supabase/server`.

```ts [server/api/me.ts]
import { serverSupabaseUser } from "#supabase/server";

export default defineEventHandler(async (event) => {
  // Get user from Supabase
  const user = await serverSupabaseUser(event);

  // Return user data to client
  return user;
});
```

2. Then, call your API route from any Vue file:

```ts [pages/index.vue]
const user = ref(null);

const fetchMe = async () => {
  user.value = await $fetch("/api/me");
};
```

## SSR support

If you're calling this route during server-side rendering (SSR), ensure that your request includes the necessary cookies, including the Supabase token. This is required to authenticate the user on the server.

Refer to the [useRequestHeaders](https://nuxt.com/docs/api/composables/use-request-headers) composable for more information.

#### Example

```ts [pages/index.vue]
const user = ref(null);

const { data } = await useFetch("/api/me", {
  headers: useRequestHeaders(["cookie"]),
});

user.value = data;
```

## Summary

- Use `serverSupabaseUser` to retrieve Supabase user data on server routes.
- Be sure to pass cookies for SSR when fetching user data on the server.
- Refer to the Nitro and data-fetching documentation for a deeper understanding.
