::callout{icon="i-heroicons-light-bulb"}
**Note:** This function is designed to work only in server routes; there is no Vue composable equivalent.
::

It functions similarly to `serverSupabaseClient`, but designed to work in server routes.

## How to use

1. Define your server route and just import the `serverSupabaseClient` from `#supabase/server`.

::callout{icon="i-heroicons-light-bulb"}
Be careful, serverSupabaseClient is returning a promise.
::

```ts [server/api/libraries.ts]
import { serverSupabaseClient } from "#supabase/server";

export default eventHandler(async (event) => {
  const client = await serverSupabaseClient(event);

  const { data } = await client.from("libraries").select("*");

  return { libraries: data };
});
```

2. Then, call your API route from any Vue file:

```ts [pages/index.vue]
const fetchLibrary = async () => {
  const { libraries } = await $fetch("/api/libraries");
};
```

## SSR support

Be careful, if you want to call this route on SSR, please read this section. You must send your browser cookies including your Supabase token. For more information, see [Data Fetching](https://nuxt.com/docs/getting-started/data-fetching#isomorphic-fetch-and-fetch).

```ts [pages/index.vue]
const {
  data: { libraries },
} = await useFetch("/api/libraries", {
  headers: useRequestHeaders(["cookie"]),
});
```
