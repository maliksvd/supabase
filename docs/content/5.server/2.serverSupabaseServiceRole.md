---
title: serverSupabaseServiceRole
description: "Make requests with super admin rights to the Supabase API with the serverSupabaseServiceRole service"
---

::callout{icon="i-heroicons-light-bulb"}
**Note:** This function is designed to work only in server routes; there is no Vue composable equivalent.
::

It functions similarly to `serverSupabaseClient`, but with one key difference: it provides a client with **super admin rights**, allowing it to bypass your Row Level Security (RLS) policies.

The client is initialized using the `SUPABASE_SERVICE_KEY`, which must be stored in your `.env` file.

For more details about Supabase keys, check out the [Supabase documentation](https://supabase.com/docs/guides/auth#service-role-key).

## How to use

1. Define your server route and import the `serverSupabaseServiceRole` from `#supabase/server`.

```ts [server/api/bypass-rls.ts]
import { serverSupabaseServiceRole } from "#supabase/server";

export default eventHandler(async (event) => {
  const client = serverSupabaseServiceRole(event);

  const { data } = await client.from("rls-protected-table").select();

  return { sensitiveData: data };
});
```

2. Then, call your API route from any Vue file:

```ts [pages/index.vue]
const fetchSensitiveData = async () => {
  const { sensitiveData } = await useFetch("/api/bypass-rls");
};
```
